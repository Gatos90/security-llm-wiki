# npm Security

npm vulnerability handling centers on package names, semantic version ranges, manifests, lockfiles, and transitive dependency resolution.

Useful public fields:

- Package name.
- Affected version range.
- Fixed version.
- Advisory identifiers such as CVE, GHSA, or OSV.
- Public maintainer notes and changelog links.
