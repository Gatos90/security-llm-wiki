# Open Source Advisory Sources

Public open-source vulnerability information is distributed across multiple sources. Different sources may disagree or update at different times.

Common sources:

- National Vulnerability Database for CVE metadata and scoring.
- GitHub Security Advisories for repository and package advisories.
- OSV for machine-readable open-source vulnerability records.
- CISA KEV for known exploited vulnerabilities.
- Vendor security advisories and release notes.
- Package registry advisories where available.

Normalization should preserve the source, publication date, affected range, fixed version, references, and uncertainty.
