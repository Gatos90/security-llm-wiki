# PyPI Security

PyPI vulnerability tracking should use canonical package names, affected version ranges, and fixed releases. Pay attention to renamed packages, extras, vendored code, and dependency resolver behavior.

Useful public fields:

- Package name.
- Affected and fixed versions.
- CVE, GHSA, or OSV aliases.
- Public project advisory or changelog.
- Runtime conditions when publicly documented.
